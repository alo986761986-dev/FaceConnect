# FaceConnect Windows Desktop App Build Guide

## Prerequisites
- Node.js 18+ installed
- Yarn package manager
- Windows 10/11 (for building Windows executables)

## Quick Build (on Windows)

1. **Clone/Download the project**

2. **Install dependencies**
   ```bash
   cd frontend
   yarn install
   ```

3. **Build the Windows installer**
   ```bash
   yarn electron:build
   ```

4. **Output files will be in:**
   - `frontend/dist/FaceConnect Setup 2.5.0.exe` - Windows installer
   - `frontend/dist/win-unpacked/` - Portable version

## Build Options

### Standard Installer (Recommended)
```bash
yarn electron:build
```
Creates `FaceConnect Setup X.X.X.exe` with:
- Installation wizard
- Desktop shortcut
- Start Menu entry
- Uninstaller

### Portable Version
```bash
yarn electron:build:portable
```
Creates a standalone `.exe` that runs without installation.

### Development Mode
```bash
yarn electron:dev
```
Runs Electron with hot reload for development.

## Auto-Update Configuration

The app checks for updates automatically. To enable:

1. Host your releases on a server (GitHub Releases, S3, etc.)

2. Update `package.json` publish config:
   ```json
   "publish": {
     "provider": "github",
     "owner": "your-username",
     "repo": "faceconnect"
   }
   ```

3. Or use generic hosting:
   ```json
   "publish": {
     "provider": "generic",
     "url": "https://your-domain.com/updates"
   }
   ```

4. Upload these files to your update server:
   - `FaceConnect Setup X.X.X.exe`
   - `latest.yml` (auto-generated by electron-builder)

## Troubleshooting

### Build fails on non-Windows
electron-builder needs wine to build Windows executables on Linux/Mac:
```bash
# Ubuntu/Debian
sudo apt install wine64

# Mac
brew install wine-stable
```

### Icon issues
Ensure icon.ico exists in `public/icons/` with proper multi-resolution format.

### Auto-update not working
- Check update server is accessible
- Verify `latest.yml` is properly formatted
- Check app has network access

## Code Signing (Optional but Recommended)

For production distribution:

1. Purchase a code signing certificate
2. Configure in `package.json`:
   ```json
   "win": {
     "certificateFile": "path/to/cert.pfx",
     "certificatePassword": "password"
   }
   ```

Without signing, Windows may show SmartScreen warnings.
